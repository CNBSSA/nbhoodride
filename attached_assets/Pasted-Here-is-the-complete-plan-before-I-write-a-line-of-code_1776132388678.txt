Implementation Plan
What's being built
Mode 3 — Multi-Stop (You Pay)

Rider adds 1–3 pickup stops (e.g. pick me up, then pick up my friend, then pick up my sister)
All pickups share ONE final destination
Fare = full-route distance (stop1→stop2→stop3→destination), single payment by organizer
Mode 4 — Shared Schedule (Code-Based)

Rider books their own ride, receives a PG-XXXXXX schedule code
They share the code with up to 2 other riders (copy/SMS/WhatsApp, manually)
Each joiner enters their own pickup AND their own destination (independent trips)
Discount logic: 30% off for EVERYONE (including organizer) the moment 1 person joins. If nobody joins → full price for organizer
Join window: stays open until driver accepts the group
Driver earns combined discounted fares (still more profitable than a single solo ride)
Existing auto-match: untouched, still works side by side

Data model
New table ride_groups: scheduleCode (unique, "PG-XXXXXX"), organizerId, groupType ('multi_stop' | 'shared_schedule'), sharedDestination (for multi_stop), maxSlots=3, filledSlots, status ('open'|'active'|'completed'|'cancelled'), driverId, discountActive

New fields on rides: groupId (FK → ride_groups), rideType (default 'solo'), pickupStops (JSON array for multi_stop), originalFare, discountAmount

Payment timing: Virtual card deduction happens at driver acceptance, not booking. At that point the server checks group size, applies discount if ≥2 riders, then deducts each rider's adjusted fare.

API endpoints added
Method	Path	Purpose
POST	/api/rides/multi-stop	Create multi-stop ride (1 payer)
POST	/api/rides/create-shared-schedule	Create shared schedule, returns code
GET	/api/rides/schedule/:code	Preview a schedule (slots, time) before joining
POST	/api/rides/join-schedule	Join someone's schedule with a code
Modified: POST /api/driver/rides/:rideId/accept — when accepting a grouped ride, automatically closes the group and applies discount to all rides if ≥2 riders.

Frontend (3 new components, 2 modified)
MultiStopBookingSheet.tsx — inline bottom sheet with 4 steps: add stops → shared destination → pick driver → confirm
SharedScheduleSheet.tsx — inline bottom sheet: your pickup + destination → pick driver → confirm → show code prominently with copy button
JoinScheduleModal.tsx — enter code → preview group → enter your pickup + destination → confirm (shows discounted fare)
RiderDashboard.tsx: idle panel gets two new icon buttons — "Multi-stop" and "Join Schedule"

IncomingRideRequest.tsx: shows "Group Ride" badge, lists all pickups, shows combined earnings
